Introduction

This is an opinionated guide for building hybrid mobile applications. It provides developer insights and solutions targeted at developers trying to construct, test, and deploy mobile/tablet hybrid applications.

What is a hybrid mobile/tablet application

Wikipedia offers a good definition of a hybrid mobile/tablet application (altered slightly for clarity):

"Hybrid apps, like native apps, run on the device, and are written with web technologies (HTML, CSS and JavaScript). Hybrid apps run inside a native container (aka webview), and leverage the deviceâ€™s browser engine (but not the browser) to render the HTML and process the JavaScript locally. A web-to-native abstraction layer (e.g. cordova) enables access to device capabilities that are not accessible in Mobile/Tablet Web applications, such as the accelerometer, camera and local storage."

Who should read this

You are front-end engineer who has experience developing applications in HTML, CSS, and JavaScript (e.g. SPA applications) that wants to use these skills to build a hybrid native mobile/tablet application and are in need of some direction/opinions.

1. The Basics

1.2 Remember why you are doing hybrid development

Built and packaged correctly, the goal of a hybrid app is to be virtually indistinguishable to end-users from a native SDK app. The advantage of building apps with HTML, CSS, and JavaScript is that developers can maintain a single code base to target all major app platforms (ios, android, windows phone). Your team can focus on HTML, CSS, and JavaScript development rather than being average practitioners of Objective-C, Java, .NET and the myriad of other developer skills need for cross-platform mobile app development.

1.3 Define targeted mobile/tablet platforms/versions

Hybrid apps run in a webview, which is like a tab in a web browser. Make sure to define upfront the mobile/tablet platform/versions in order to take note of the HTML5 (aka JavaScript API's), CSS, and JavaScript standards that are support by the mobile platform.

Unlike browser development for the desktop it's more than likely that your application will be run by a newer, modern, browser engine that supports things like ES5, CORS, and Web Sockets.

1.4 Use a mobile widget toolkit with application logic

A native mobile/tablet application is the ideal candidate for a widget toolkit. What you want to find is something like Kendo UI mobile, where the widgets have supporting application tools (MVVM, router, layout, view) to aid in the production of a JS/SPA application.
